Code Log 5: Images

There are multiple ways of putting images on a web page. Using HTML to put an image using the <img> tag is different from using the CSS property background-image. The <img> tag embeds an image onto the web page using the src attribute, acting like an element on the page, similar to a section of text. The <img>> tag in HTML can not have any child content either, as it doesn't have a closing tag because it is often formatted as follows: <img src="tree.jpg">.

The CSS property background-image puts an image onto the webpage as a background image of an element from the HTML, like a

tag. An image embedded by the background-image property is formatted like this: background-image: url("tree.jpg"); An image put on the webpage using the background-image property can be manipulated by using other CSS properties like background-position which can choose how the background image is positioned on the page, like using background-position: center; to center the image in the element it is embedded in. Another property that can manipulate a background-image is background-size, which allows you to adjust the image's size within the element. For example, using background-size: cover; will cover the element with the entire image, rather than displaying a section or zoomed-in portion of the image.

In a bitmap image, the dimensions are the width and the height of the image measured in pixels, while resolution is the ratio of pixels used in an image measured in pixels per inch.
